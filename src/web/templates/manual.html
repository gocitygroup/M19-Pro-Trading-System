1) What this system is

G-SignalX M19 Trading System is a trading operations dashboard designed to help traders monitor positions, manage exits, and standardize session-based execution. You can use it with any signal source (manual chart analysis, indicator alerts, paid signals, or proprietary models).

Important: This system supports multiple instrument types depending on what your broker exposes in MT5 (e.g., FX pairs, metals, indices, crypto CFDs). Your risk model must be aligned to the instrument’s volatility.

2) Safety-first setup

  • Start on a demo account to validate execution, spreads, and risk sizing.
  • Test each change one at a time (volume, trailing, check intervals) and log outcomes.
  • Graduation rule: go live only after 2–4 weeks of stable behavior.

  • MT5 is open and connected.
  • Dashboard shows Connected status.
  • Profit Monitor / bots are running (if you use automation).
  • High-impact news is checked and respected.
  • Max daily loss and max daily profit objectives are defined.

Use this as a safe baseline; adjust to your broker + instrument volatility:

Trading Bot
- volume: 0.01 to 0.05 (small lots)
- session_check_interval: 2 to 10 sec

Profit Monitor
- min_profit_percent: 0.3 to 1.0%
- trailing_stop_percent: 0.2 to 0.6%
- check_interval: 10 to 60 sec (faster for scalps, slower for swings)

3) Daily workflow (repeatable operations)

Pre-session
  • Define session bias (Buy/Sell/Neutral).
  • Confirm spreads and liquidity.
  • Set risk limits (daily loss cap).
During session
  • Monitor open positions and exposure.
  • Use partial closes + trailing to reduce emotion.
  • Respect news and volatility spikes.
Post-session
  • Review P&L + screenshots/notes.
  • Tag mistakes (overtrade, revenge, late entry).
  • Backup working settings.

6) Risk management (non-negotiables)

  • Risk per trade: define a fixed rule (e.g., 0.25%–1.0% of equity).
  • Daily loss limit: stop trading at your cap (system discipline beats “hope”).
  • Correlation control: avoid stacking highly correlated instruments in the same direction.
  • News discipline: high-impact events can invalidate technical logic instantly.
If you’re unsure, reduce size first. Most failures come from oversizing + overtrading, not from missing an entry.

7) Troubleshooting

  • Confirm the web server is running.
  • Refresh the page and try Reconnect.
  • Verify firewall rules allow the configured port.

  • Confirm you have open positions in MT5 on the same account.
  • Click Refresh Data and wait 10–20 seconds.
  • Ensure the MT5 Python integration is available on this machine.

  • Check symbol availability, market hours, and trading permissions.
  • Reduce lot size and confirm min/max lot constraints.
  • Widen tolerance if spreads are elevated.
  • Review logs for broker error messages.

8) Glossary

Session bias: Direction intent for a time window (Buy/Sell/Neutral).
Trailing stop (%): A dynamic exit that locks profit as price moves.
Partial close: Close part of a position to secure gains and reduce risk.
Correlation: Instruments moving together, amplifying risk.
